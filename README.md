# TicketApp: Микросервисная система продажи билетов

## Описание проекта

TicketApp — это веб-приложение для продажи билетов на концерты, разработанное на основе микросервисной архитектуры. Проект демонстрирует взаимодействие нескольких независимых сервисов, асинхронную обработку задач и централизованное управление через API-шлюз.

## Архитектура

Проект состоит из следующих компонентов:

- **Auth Service** (`auth-service`): Отвечает за регистрацию, аутентификацию пользователей и выдачу JWT-токенов.
- **Admin Service** (`admin-service`): Основной бэкенд-сервис, который управляет данными о концертах, артистах, билетах и бронированиях. Также предоставляет админ-панель для управления контентом.
- **Concert Service** (`concert-service`): Фронтенд-сервис, который отображает веб-страницы для пользователей (афиша, детали концерта, бронирования) и взаимодействует с другими сервисами через API.
- **PostgreSQL** (`postgres`): Реляционная база данных для хранения всей информации.
- **RabbitMQ** (`rabbitmq`): Брокер сообщений для асинхронной обработки задач, таких как создание бронирований.
- **Nginx** (`nginx`): API-шлюз и обратный прокси, который распределяет все входящие запросы между микросервисами.

## Технологии

- **Бэкенд**: Python, Flask
- **База данных**: PostgreSQL
- **Очередь сообщений**: RabbitMQ
- **API-шлюз**: Nginx
- **Контейнеризация**: Docker, Docker Compose
- **Документация API**: Swagger (Flasgger)

## Запуск проекта

1.  **Клонируйте репозиторий**:
    ```bash
    git clone <URL вашего репозитория>
    cd TicketApp
    ```

2.  **Убедитесь, что у вас установлен Docker и Docker Compose.**

3.  **Запустите все сервисы** с помощью Docker Compose:
    ```bash
    docker-compose up --build -d
    ```

4.  **Приложение будет доступно** по адресу [http://localhost](http://localhost).

## Основные эндпоинты

- **Главная страница (Афиша)**: `http://localhost/`
- **Админ-панель**: `http://localhost/admin/dashboard` (требуется авторизация с правами администратора)
- **API документация (Swagger)**:
  - **Auth Service**: `http://localhost/auth/apidocs/`
  - **Admin Service**: `http://localhost/apidocs/`
  - **Concert Service**: `http://localhost/concert/apidocs/`
